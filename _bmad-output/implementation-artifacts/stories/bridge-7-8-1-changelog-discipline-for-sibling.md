**Decisions made:**
- Documented the bridge-6-7-3 contract change under CHANGELOG `[Unreleased]/Изменено` (not Добавлено): the user-visible delta is a *changed* installer contract (hook-count 2→3), even though the mechanism was an added pre-commit hook. Keep-a-Changelog semantics favour "Changed" for a contract bump downstream consumers must react to.
- Placed the AC2 discipline note in `_docs/RELEASE_PROCESS.md` (new "Дисциплина installer-contract bump'ов" section) rather than CONTRIBUTING.md, because RELEASE_PROCESS already owns the CHANGELOG-backfill flow and the husky pre-commit reminder callout — the rule sits next to the machinery it governs.
- Authored both entries in Russian to match the sibling repo's established CHANGELOG/RELEASE_PROCESS language convention (surgical consistency), despite the IQ-ME doc-output default being English — this is a sibling-repo artifact.

**Alternatives considered:**
- Adding the discipline note to CONTRIBUTING.md — rejected: CONTRIBUTING has no release/CHANGELOG section today, so the note would float context-free.
