# Story bridge-7-8-3-replace-legacy-exempt-central: Replace LEGACY_EXEMPT central list with per-spec frontmatter flag

Status: backlog

## Story

lint-spec-carry-forward.mjs LEGACY_EXEMPT array (tools side) flags legacy specs that pre-date the Carry-forward enforcement. Each retroactive remediation requires editing the array centrally — brittle as new stragglers surface. Replace with a per-spec frontmatter field lint-exempt-carry-forward set to true; lint reads the frontmatter and skips the spec if set. Migration: one-time pass converts current LEGACY_EXEMPT entries to frontmatter flags + removes the array. Same lint behavior, distributed maintenance, no central-edit gate. Time-box half a day.

## Sources (deferred from)

- `bridge-6-7-3-restore-carry-forward-lessons` (kind=completion-note) — Specialist Self-Review Areas-of-uncertainty bullet 3 — Carry-forward section lint exempt-list LEGACY_EXEMPT is brittle, requires manual maintenance

## Acceptance Criteria

1. `tools/lint-spec-carry-forward.mjs` reads a per-spec frontmatter field `lint-exempt-carry-forward: true` and skips the Carry-forward-section check for that spec; the central `LEGACY_EXEMPT` Set is **removed**.
2. **One-time migration:** every spec currently listed in `LEGACY_EXEMPT` (epic-1..3, parts of 4..5, `bridge-1-2-1`, `bridge-6-7-3`, etc.) gets `lint-exempt-carry-forward: true` added to its frontmatter, so the lint's exempt set is byte-identical before/after the change (same specs exempt, same NEW-spec enforcement).
3. Adding/removing an exemption no longer requires a central edit to the lint script — it is a per-file frontmatter change (distributed maintenance, no central-edit gate).
4. Lint still **fails** for any NEW spec that lacks the Carry-forward section AND lacks the `lint-exempt-carry-forward` flag.
