# Story bridge-7-8-3-replace-legacy-exempt-central: Replace LEGACY_EXEMPT central list with per-spec frontmatter flag

## Story

lint-spec-carry-forward.mjs LEGACY_EXEMPT array (tools side) flags legacy specs that pre-date the Carry-forward enforcement. Each retroactive remediation requires editing the array centrally — brittle as new stragglers surface. Replace with a per-spec frontmatter field lint-exempt-carry-forward set to true; lint reads the frontmatter and skips the spec if set. Migration: one-time pass converts current LEGACY_EXEMPT entries to frontmatter flags + removes the array. Same lint behavior, distributed maintenance, no central-edit gate. Time-box half a day.

## Specialist Self-Review

**Decisions made:**
- Frontmatter flag read via a narrow regex (`/^lint-exempt-carry-forward:[ \t]*true[ \t]*$/im`) scoped to the leading frontmatter block only (sliced at the first `\n---`), so a `lint-exempt-carry-forward` mention in spec body prose can't accidentally exempt. Flag must be exactly `true`; `false`/absent does not exempt (AC4).
- Migration adds the flag as the **first line after the opening `---`** of each former-exempt spec — minimal, frontmatter-local edit; specs without frontmatter would get a fresh block (none hit that path — all 39 had frontmatter).
- `--ignore-old` retained as an accepted **no-op** (Makefile/CI passes it) rather than removed, to avoid breaking the existing invocation; exemption is now frontmatter-driven regardless of the flag.

**Alternatives considered:**
- Keeping `--ignore-old` meaningful (e.g. a different gate) — rejected: with the central list gone it has nothing left to gate; a silent no-op with a doc note is least-surprise for CI.
- Adding sections/flags to the in-flight bridge-7-8-* specs to make the lint green — rejected as scope-creep (see uncertainty).

**Framework gotchas avoided:**
- All 58 story specs are integrity class-A. The migration drifted 39 of them; re-recorded via one batched `tds integrity record --as=engineer` (lesson-2026-05-19-001 / lesson-2026-05-20-007) — `tds integrity verify` is back to 162/0.
- The frozen bridge-6-7-3 test (`lint-spec-carry-forward-coverage.test.mjs`, class-A) was **not** touched; new behavior went into a separate new test file, so no frozen-test edit / re-freeze was needed.
