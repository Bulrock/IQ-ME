## Specialist Self-Review

**Decisions made:**
- Consolidated the escape/interpolation duplication (escape helper copied 7× across 6 modules in two variants; `fmt` 2×) into a single leaf module `src/assessment/html-util.js` exporting `escapeText` (3-replace, text content), `escapeAttr` (4-replace, attribute values; reuses escapeText), `fmt`. Wired consumers with **aliased imports** (`escapeAttr as E`, `escapeAttr as esc`, etc.) so every call-site is unchanged → output is byte-identical, zero behavior change.
- Re-pinned the ceiling to **48128 (47 KB)** = post-dedup measured size (47223) + a deliberately tight ~2% margin, with a rationale that supersedes the five ad-hoc Epic-6 raises. Chose tight-on-purpose (the lesson is the budget "decayed toward a rubber stamp") so the next raise needs explicit justification; epic-7 i18n weight raises it *with* rationale when it lands.

**Alternatives considered:**
- Extracting to a heavily-commented/multi-line shared module — measured and **rejected**: my first cut *raised* app-modules-bytes to 47647 because the per-file byte budget counts the shared file + imports. Compacting the util (single-line, minimal comment, escapeAttr reusing escapeText) flipped it to a 253-byte net reduction.
- Unifying everyone onto one 4-replace escape — rejected: would change output for text-content strings containing `"` (3-replace → 4-replace), a real behavior delta. Kept both variants mapped exactly.

**Framework gotchas avoided:**
- Per-file byte budgets make naive "extract to shared util" *increase* total bytes; the win here is structural (single escape source, no 3-vs-4-replace drift) with a modest absolute delta — framed honestly as principled, not a token trim.
- Budget test (`cognitive-load-budget.test.mjs`, frozen, owned by 6-5) edited via the unfreeze-window ceremony against the owning story; 6-5 stayed `done` (auto-flip only fires from tests-approved/tests-drafting).
