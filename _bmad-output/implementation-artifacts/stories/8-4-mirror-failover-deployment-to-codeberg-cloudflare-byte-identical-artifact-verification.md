# Story 8.4: Mirror failover deployment to Codeberg/Cloudflare (byte-identical artifact verification)

## Story

As a **Russia-based user whose ISP has blocked GitHub Pages (Risk #8)**,
I want **the Codeberg Pages (or Cloudflare Pages) mirror to host a byte-identical artifact accessible at a documented secondary domain — the mirror triggered by the same `release.yml` workflow as the canonical, with a post-deploy byte-identical-artifact assertion**,
so that **the Russia hosting block (Risk #8 / NFR17 mirror-readiness) is mitigated WITHOUT a JS-based detection redirect (PRD §Mirror Strategy — "no automatic redirect, no JS-based detection")**.

## Acceptance Criteria

> **Epic-8 dev-phase decision (infra-now) — no live mirror deploy, no live fetch/SHA compare, no Codeberg/Cloudflare account, no live Playwright-against-mirror run.** Same posture as Story 8.1 / Story 8.2 / Story 8.3 / Story 7.8. The mirror is deployed and the byte-identical fetch/compare + the Playwright-against-mirror happy-path first fire against live hosts at launch (Epic 10), once the Codeberg repo / Cloudflare project + DNS + the mirror secrets exist and a real `app-v*`/`corpus-v*` tag is pushed. The dev-phase deliverable is: (a) `release.yml` gains a **structurally-correct `deploy-to-mirror` job** (pushes the same `dist/` artifact to the Codeberg/Cloudflare endpoint) + a **byte-identical-artifact verification step** (fetch the same path from canonical + mirror, SHA256-compare the four named artifacts, fail on mismatch); (b) every live call (the mirror push, the fetch/SHA compare, the Playwright-against-mirror run) is gated **INERT** behind a launch repo var — mirror the 8.2/8.3 `vars.IQME_LIVE_*` pattern (use a parallel `vars.IQME_LIVE_MIRROR`); (c) `README.md` documents the canonical URL, the mirror URL, the manual-failover trigger-policy, and explicitly states no automatic redirect; (d) a **structural contract test** (the test-author-phase tasks below). NO mirror push, NO live `curl`/SHA256 compare, NO live Playwright run executes in dev. The structural contract is verified by tests parsing `release.yml` + `README.md` + the `src/index.html` chrome-footer block. **No fabricated SHA256 hashes, no fabricated mirror URL responses** (lesson-2026-06-04-002 — reference the real endpoints + an explicit launch-gated posture, never fabricate specifics).
>
> **Scope boundary.** This story OWNS the `deploy-to-mirror` activation. It REPLACES the `release.yml` Story-8.4 deferred marker comment (`# --- Deferred to Story 8.4: Codeberg / Cloudflare Pages byte-identical mirror … ---`, ~lines 16-17 header + ~lines 170-172 tail) with the real job. It does NOT touch the Story 8.1 app-release/corpus-release jobs or the Story 8.2 archival steps (Zenodo/IA/SH stay byte-identical). The **live** mirror deploy + the **live** Playwright-against-mirror run are launch-time (Epic 10) — dev-phase asserts the workflow STEPS exist (structural), not a live mirror run. The weekly `scheduled.yml` `mirror-parity-check` (Story 8.3) is **self-gating** and already lands cleanly before this mirror exists — this story does NOT modify `scheduled.yml`.
