# Story 14.11: Rendered visual-regression, contrast, reduced-motion and print verification suite

## Story

As a **psychometrician-minded reviewer auditing the Aurora redesign**,
I want **a browser-rendered verification suite that proves the deep-navy spatial backdrop, glass legibility, question/answer scale parity, focus visibility, reduced-motion behaviour and print output actually render correctly — not merely that the CSS source parses**,
so that **the Epic 14 visual changes are guarded against silent regression by what the user sees, since structural CSS-source assertions cannot catch a backdrop that composites flat or an option icon that mismatches its matrix cell**.

## Acceptance Criteria

1. **toHaveScreenshot per route × 7 viewports × light/dark (PR-30 / design §5).** The rendered spec drives the seeded happy-path via `window.__IQME_TEST__` (the [trust-verification.spec.mjs](../../../tests/playwright/trust-verification.spec.mjs) `?test=1` + [tools/dev-server.mjs](../../../tools/dev-server.mjs) template) and calls `await expect(page).toHaveScreenshot()` per route — landing, consent, item-runner, result pre-reveal, revealed score-panel, top-decile + bottom-decile tail-scenes, and ≥1 methodology page — at the approved viewports 320/375/414/768/1024/1280/1440, in both `light` and `dark`. The spec runs with `maxDiffPixelRatio` in the documented ~0.01–0.02 (1–2%) range to tolerate font-hinting jitter without masking composition drift. The frozen Epic 11/13 DOM contracts (`.landing`, the co-equal `.score-panel` Percentile/IQ/Range triplet) are exercised but unmodified, and the run stays zero-third-party (NFR6) — `@playwright/test` + `@axe-core/playwright` are installed only at CI time, never bundled.

2. **±5% computed-geometry matrix-cell-to-option-icon scale parity (PR-30 / design §5).** The suite navigates to `test`, waits for `.item-runner`, and reads the live `getBoundingClientRect()` of the rendered matrix cell (matrix image width ÷ the `data-grid-cols` attribute Story 14.6 added) versus an option figure at each approved viewport. The computed option-icon width is asserted within ±5% of the computed matrix-cell size, and both retain `aspect-ratio: 1 / 1` (no distortion). The assertion reads rendered computed geometry (not CSS source) and preserves the frozen item-runner DOM.

3. **No clipping / no distortion across the seven widths (PR-30 / design §5, NFR1).** Iterating the seven widths, the suite asserts each rendered matrix and option image bbox stays within its container and within the viewport, and no horizontal scroll appears (`scrollWidth <= clientWidth + 1`, mirroring [trust-verification.spec.mjs](../../../tests/playwright/trust-verification.spec.mjs)) across the desktop/mobile branch, preserving NFR1 no-horizontal-scroll and leaving the frozen item-runner contract intact.

4. **Focus-visible + `:checked` over the Aurora backdrop (PR-30 / WCAG 2.2 AA).** The suite tabs through the options keyboard-only, asserts a visible focus indicator on the focused control (computed outline differs measurably from the unfocused state), the `:checked` selected-answer border renders over the Aurora backdrop, and captures a `toHaveScreenshot` of the focused option. Keyboard reachability of every option is preserved and the run honours WCAG 2.2 AA focus-appearance with zero-third-party (NFR6).

5. **Reduced-motion stable static end-state (PR-30 / design §4.2).** The suite re-runs the route screenshots under `await page.emulateMedia({ reducedMotion: 'reduce' })` and asserts the Aurora-animated surfaces settle to their static end-state — a stable `toHaveScreenshot` under reduced-motion that matches across two passes (proving the reduced-motion branch renders rather than animating) — on landing and result, without breaking the deep-navy backdrop contrast or the co-equal triplet, preserving the byte-stable build (NFR21) since these are test-only baselines.

6. **Print / PDF document leg (PR-30 / design §3.3).** The suite seeds a result session and emulates print media (`emulateMedia({ media: 'print' })`) and exercises `page.pdf()`, then asserts the printed result contains the co-equal Percentile/IQ-scale/Range triplet, the difficulty band, and the disclaimer rendered open, with glass surfaces and CTAs dropped, captured as a committed print baseline (baseline generation deferred to CI per scope). The print path stays local-only with no telemetry (NFR6) and the co-equal triplet invariant is preserved in the print rendering.

7. **axe-core AA contrast over the rendered gradient backdrop (PR-30 / NFR12, WCAG 2.2 AA).** The suite extends the [trust-a11y.spec.mjs](../../../tests/a11y/trust-a11y.spec.mjs) AA-only axe scoping (`disableRules(["color-contrast-enhanced"])`) to assert WCAG 2.2 AA contrast for body text, action controls, focus indicators, and selected-answer borders specifically over the gradient backdrop on each Aurora surface, reporting zero contrast violations against the rendered gradient (not a flat token), in both `[data-theme]` palettes via `prefers-color-scheme`, without weakening the existing AA assertion.

8. **CI wiring — CODE-COMPLETE, job DORMANT (PR-30 / epics.md Decision 4).** The authoring host is darwin; CI is `ubuntu-latest`. Playwright names snapshots per-platform (`*-linux.png`), which cannot be produced on darwin, so this story authors the FULL rendered spec but commits NO PNG baselines and creates NO `*-snapshots/` directory. The [pr-checks.yml](../../../.github/workflows/pr-checks.yml) `visual-regression` job therefore STAYS `if: false`; its `# Activates in Epic 14 (Story 14.11)` comment is updated to record that the spec is authored/code-complete and that final activation is pending the `ubuntu-latest` baseline bootstrap. The job stays a DEFERRED job in [ci-matrix.test.mjs](../../../tests/scaffold/ci-matrix.test.mjs)'s `ALL_JOBS` (NOT promoted into `EPIC_1_ACTIVE`), keeping AC-1/AC-2/AC-3 of the ci-matrix discipline test green. [docs/required-ci-checks.md](../../../docs/required-ci-checks.md) gains a row for the job under the Accessibility/Playwright section MARKED deferred/dormant ("activates once ubuntu-latest baselines are committed"), not asserted as a currently-required check. No other CI job is added.

## Dev Notes

### Implementation strategy
- **This story authors a RENDERED test suite — it is test-only.** No production JS/CSS/markup is edited; the Aurora surfaces are already built by Stories 14.3–14.10. The deliverable is the extended Playwright spec (the rendered legs the AC enumerates) + a scaffold source-text guard that runs in `make test` and is the local verification, + the CI wiring update (comment + docs row).
- **The seeded harness is the driving template.** Every leg drives the SPA via `?test=1` + `window.__IQME_TEST__` ([test-hook.js](../../../src/assessment/test-hook.js): `resetState` / `setSeed` / `recordResponse` / `navigate`) against `tools/dev-server.mjs`, exactly like [trust-verification.spec.mjs](../../../tests/playwright/trust-verification.spec.mjs) and [trust-a11y.spec.mjs](../../../tests/a11y/trust-a11y.spec.mjs). The tail variant is selected by the response bias (all-1 → top-decile, all-0 → bottom-decile, alternating → mid band), which drives `selectTailScene()` → the `.score-panel--top-decile` / `--bottom-decile` modifier in [result.js](../../../src/assessment/result.js). The reveal is `.rs-show` ("Show me") → `.score-panel`.
- **The ±5% parity leg reads RENDERED geometry, not CSS source.** The matrix CELL size is the rendered `img.item-runner__image` width ÷ the `data-grid-cols` attribute (Story 14.6's additive attribute on `.item-runner`), compared against the `.item-runner__option-figure` / `img.item-runner__option-image` bbox. The matrix is height-bound (`max-height: 42vh` desktop / `30vh` mobile) and options are `width: clamp(...)`, with NO runtime link — so this is the leg that catches a divergence at any of the 7 widths. Both bboxes are asserted square (aspect-ratio:1/1 within a sub-pixel tolerance).
- **axe-core AA-only scoping mirrors trust-a11y.** The contrast leg uses `AxeBuilder().withTags([... wcag2aa ...]).disableRules(["color-contrast-enhanced"])` so it enforces exactly the WCAG 2.2 AA 4.5:1 / non-text bar over the RENDERED gradient (not the AAA 7:1 rule), in both `colorScheme` palettes, over landing + item-runner + revealed result.

### CODE-COMPLETE, job DORMANT — the ubuntu-latest baseline-bootstrap follow-up
- **Why the job stays `if: false`.** Playwright's `toHaveScreenshot` names snapshots per-platform (`<name>-<project>-linux.png`). The CI runner is `ubuntu-latest`, so the committed baselines must be `*-linux.png`, which **cannot be generated on the darwin authoring host** (they would be `*-darwin.png` and would not match the CI platform). Committing darwin baselines, or flipping `if: false` -> on without any baselines, would make the gate fail on the first run. The maintainer's decision is therefore: **author the FULL spec now (CODE-COMPLETE), keep the job DORMANT.**
