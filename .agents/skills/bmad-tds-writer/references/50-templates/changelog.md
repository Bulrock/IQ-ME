## Notes for writer

- **Past tense** — entries описывают what shipped. («Added X», not «adds X».)
- **User-facing language.** «Fixed login bug» не «Patched authentication race condition в auth.py:42».
- **Unreleased section на top** — always present; entries accumulate during dev cycle, get версионировано при release.
- **Group by category** — Added / Changed / Deprecated / Removed / Fixed / Security.
- **Link to migration guide** for breaking changes. Don't bury migration steps в CHANGELOG entry.
- **Link compare URLs** at bottom — auto-generated by tag references.
- **Don't dump every commit.** CHANGELOG curates, не lists. PRs document themselves; CHANGELOG explains к user.
- **Stakeholder version** — separate file `RELEASE_NOTES.md` для customer-facing (Persona 5). Less technical, outcome-focused.
